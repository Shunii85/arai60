# 計算時間の概算

計算量から、最悪の計算時間を概算してみる。他の人のコメントにもあった。

https://github.com/Yuto729/LeetCode_arai60/pull/16#discussion_r2602118324

> アルゴリズムを実装する前に、まず時間計算量を求め、その計算量から実行時間を概算することをおすすめします。
実行時間を推定する際は、計算量の式にデータの最大サイズを代入して「おおよその計算ステップ数」を算出してください。
その上で、このステップ数を「言語ごとの 1 秒あたりに処理可能なおおよその計算ステップ数」で割ると、おおよその実行時間（秒）が求まります。
> - C++： 約 1～10 億ステップ/秒 <br/>
  CPU の周波数が数 GHz 程度であるため、IPC（Instructions per Cycle）を 1 と仮定すると、1 秒間に数十億命令を実行できます。
  C++ はオーバーヘッドが小さく、1 ステップを数命令〜数十命令で実行できるため、おおよそ 1～10 億ステップ/秒が目安となります。

日本の数字って10^4ごとに単位があがるの、違ってややこしい。
Gは10^9だから、10^8で億だから、10億。

ここでいう、サイクルとステップの違いは何？
- サイクルはクロックサイクルのこと。
  - ここではHzは、一秒間に何クロックサイクル回るかどうか。
- ステップ数は、計算量を考えるときのざっくりとしたやりたいことの単位。およそと書いてあるからちょっと違うかも。
  for文N買いまわすなら、Nステップ的な。
  そーとにはNlogNステップくらいかかりますよ。

  計算量に代入して求まるのがだいたいのステップ数。
- 結局C++はアセンブリ、機械語とコンパイルされるわけで、実際その1ステップは何命令でできるの？というと、数命令から数十命令。
- よって、数十命令使って1ステップ実行するなら1億ステップ。数命令で済むなら10億ステップ程度。なるほどね。あたりまえだけど1命令で1ステップ実行できるならそちらのほうが優秀。

サイクル(= 周波数, 1秒間) => IPC(1サイクルで実行できる命令数) => 命令 => ステップ (何命令で1ステップ実行できるか)

逆順でたどるとわかる。
