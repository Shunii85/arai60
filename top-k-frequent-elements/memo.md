# step1

手作業で考えてみる。

とりあえず例として、
> 10万枚くらいの大量のTシャツがあって、それぞれ番号が振られている。
そこから最も在庫を抱えているk種類のTシャツが何かを探し出してきてほしい。

こんなところだろうか。候補として真っ先に上がるのは、ホワイトボードに番号ごとに何枚のTシャツが現在数えられているのか
メモしておく方法。だけどこれだとメモが大量過ぎて難しそう。
メモを残すのはまだ簡単でも、集計してk種類を探し出すのはめんどくさそう。

ここで他の問題解決の方法が思い浮かばずに断念。

とりあえずHeapを使ってやってみる。 前回は最小ヒープを使ったけれど、最大ヒープを使えば、上からK回取り出すだけで大丈夫そう。
ヒープの構築にO(N)、取り出し(削除)にO(KlogN)がかかったとして、O(NlogN)で問題が解けそう。
領域計算量はO(N)かな。

何かおかしいことをやっていた。頻度なので、その数字自体で並び替えを行っていては何の意味もない。忘れよう

方針は理解できて、
まず要素の数を数え上げて保存する。
在庫の数をもとにならべかえる。
この並べ替えの役割が、Heap
昇順、つまりHeapのtopは最も小さくなるようにして、常にsizeがk個になるようにすることで、
topがk番目の頻度で出現する要素になる。こうすると、Heapに詰められているのはk番目までの、k個の要素になる。

  関数オブジェクトを簡単に宣言したものをラムダ式というらしい。

  そもそも、関数オブジェクトの利点は？

  https://learn.microsoft.com/ja-jp/cpp/standard-library/function-objects-in-the-stl?view=msvc-170

  > 関数オブジェクトは、通常の関数呼び出しよりも 2 つの主な利点を提供します。 1 つ目に、関数オブジェクトには状態を含めることができます。 2 つ目に、関数オブジェクトは型であるため、テンプレート パラメーターとして使用できます

  こういうもの
  ```cpp
  auto plus = [](int a, int b) { return a + b; };
  ```
  ↓
  一時的にこういう定義がなされて、それで使える。型はわからない。よって後述のdecltypeが必要。
  ```cpp
  struct F {
    auto operator()(int a, int b) const -> decltype(a + b)
    {
      return a + b;
    }
  };
  ```

  [&変数名]で、ラムダ式の外の変数をキャプチャできるらしい。

  今回は、inventory変数を参照キャプチャした。これで、参照として扱える。

  ```cpp
  auto comp = [&inventory](int n1, int n2) {
    return inventory[n1] > inventory[n2];
  };
  ```
