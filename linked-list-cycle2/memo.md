# step4

フロイドのアルゴリズムで考える。

前回の問題の証明から、出会うことはわかっている。

サイクルに入るまでの長さをL、サイクルの長さをC、サイクルの始まりから出会った場所の距離をdとすると、

n秒後には、S(slow pointer)はnステップ進み、そこで出会ったとすると、

n = L + d + kC (kは何周したか)

ここで、前の問題の証明から、nはCの倍数であることがわかっているので、

0 ≡ L + d (mod C)

L + d ≡ 0 (mod C)

よって、dの距離から、Lだけ進むと、一周して、サイクルの入口に戻ってくることがわかる。

よって、headから進むポインタと、出会った場所から進むポインタを2つ用意して1つずつ進めていき、
出会ったところが入口になる。(なぜなら、2つのポインタともL進むから同じ場所にたどり着く)

これで、unordered_setを使わなくても求められる。

計算量としては、ぶつかる位置を見つけるのに循環の長さ( <= N)の定数倍がかかり、
開始地点を見つけるのにも出会った場所から開始地点までの探索時間がかかるので、O(N)


でもこの定数倍というのもどこまで大きくなるんだろう？

でも、まずfastとslowの進み方から、+1ずつ距離が空いていくことがわかる。
よって、循環の長さより大きいステップはかからない。+Cになったら出会っている。

で、L < Nだからこれはわかりやすい。

O(N)になる。
